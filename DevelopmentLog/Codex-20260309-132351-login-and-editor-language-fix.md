# 登录页与编辑器语言修复记录

## 本次修复

这次主要处理了三个用户直接反馈的问题：

1. 后台已经切到中文时，登录页 `admin/index.html` 仍然保持英文。
2. 文章编辑器在 `MdEditor` CDN 依赖异常时没有可用的编辑输入区域。
3. 编辑器里的“编辑器限制说明”由于文本节点被 `<code>` 标签拆开，自动翻译没有完整生效。

## 具体改动

### 1. 登录页接入语言状态

- 在 `admin/index.html` 内新增了与后台共用的 `admin-locale` 本地存储读取逻辑。
- 登录页现在会在加载时自动读取当前语言，并同步：
  - 页面标题
  - 顶部说明
  - 表单标题与描述
  - 用户名 / 密码标签
  - 输入框占位符
  - 登录按钮文案
  - 返回站点文案
  - 错误提示文本
- 登录页新增了右上角语言切换按钮，切换后会立即刷新页面文案，并保持与后台一致。

### 2. 编辑器页增加后备编辑器

- 在 `admin/editor.html` 中，`MdEditor` 组件仍优先使用原来的 CDN 编辑器。
- 如果 CDN 编辑器没有成功加载，不再只显示一段说明文字，而是降级为一个可编辑的纯文本 `textarea`。
- 这样即使第三方编辑器资源临时不可用，文章内容仍然可以继续编辑和保存。

### 3. 编辑器限制说明改成显式双语

- 原来“编辑器限制说明”三条内容依赖公共自动翻译。
- 由于每条说明里夹杂了 `<code>`，真实 DOM 被拆成多段文本节点，自动翻译无法稳定命中整句。
- 现在这三条说明已改为在 `admin/editor.html` 中显式按语言输出，不再依赖公共词表拼接。
