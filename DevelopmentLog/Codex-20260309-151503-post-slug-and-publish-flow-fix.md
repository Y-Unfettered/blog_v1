# 文章别名与发布流修复记录

时间：2026-03-09 15:15:03

## 问题背景

本轮修复针对后台文章编辑器的两个实际问题：

1. 新增文章时，若标题为中文，编辑器会直接把中文标题写入 `slug`，导致前台文章地址出现中文路径。
2. 文章发布成功后，页面仍停留在编辑器，不会返回文章列表。

## 本次改动

### 1. 统一前后端的 slug 规则

- 更新 `admin/editor.html`
- 编辑器自动生成的 `slug` 改为只保留 `a-z`、`0-9` 和 `-`
- 手动输入 `slug` 后，在输入框失焦时会自动净化
- 中文标题无法生成可读别名时，会回退到 `post-时间戳` 或 `post-id`

- 更新 `scripts/admin-server.cjs`
- 服务端保存文章时，不再直接信任前端传入的 `slug`
- 即使前端传入中文 `slug`，服务端也会再次归一化，确保最终落盘值为 ASCII 安全格式

### 2. 发布成功后返回文章列表

- 更新 `admin/editor.html`
- “保存草稿”保持原行为，仍停留在当前编辑页
- “发布”成功后，短暂提示后自动跳转到 `/admin/posts`

### 3. 补充自动验证

- 更新 `scripts/smoke-test.cjs`
- 新增中文标题/中文别名的文章创建用例
- 校验保存后的 `slug` 必须为 ASCII 安全格式，避免后续回归

## 验证结果

已执行并通过：

- `npm run lint`
- `npm run validate:data`
- `npm run build`
- `npm run smoke:test`

## 结果说明

现在新增文章时：

- 英文标题会继续自动生成可读别名
- 中文标题不会再生成中文地址
- 文章发布成功后会返回文章列表

当前未做浏览器逐项人工回归；本次确认以构建、数据校验和 smoke test 为主。
